viewport如何影响移动端页面展示

发表于 2016-10-29   |   分类于 移动端

本文主要记录了关于viewport的一些知识.

viewport如何影响移动端页面展示

####viewport
在项目中我们经常会在css中给一个元素声明一个百分数宽度，例如div的宽度是60%，计算该元素的宽度，要知道其包含块也就是父级的宽度，如果父级的宽度未定义，就会一直往上找，直到找到html的宽度,来计算这个div的宽度，但是我一直有个疑问html元素是否有包含块？最终找到答案html的包含块就是viewport,也被称为初始包含块。这个初始包含块将css的布局限制在一定的范围内。

在pc端，viewport就是浏览器的窗口。但是在移动端有三种viewport:

1.layout viewport（布局视窗）

是提供给CSS计算的视窗，页面中如果没有设置meta的viewport，layout viewport就是各个浏览器规定的默认值，多数浏览器中的layout viewport的大小是980px，只有BlackBerry和IE的默认值是1024px。这样做主要是为了让没有做移动端适配的页面在移动端可以正常显示出来，在页面中会出现横向和纵向的滚动条。

通过 document.documentElement.clientWidth 获取

或者 document.documentElement.getBoundingClientRect().width 获取

2.visual viewport（视觉视窗）

移动设备中浏览器的可视区域，也就是手机屏幕的宽度。常见iPhone的visual viewport:

  • iPhone4 ~ iPhone5s : 320*480px
  • iPhone6 ~ iPhone6s : 375*627px
  • iPhone6p ~ iPhone6sP: 414*736px

通过 window.innerWidth 获取

当页面中没有对meta标签中的viewport属性做设置的时候，在多数浏览器中中默认layout viewport = visual viewport,未适配移动端页面缩小全部展示在浏览器中，没有横向滚动条。这种情况下要想看清楚页面中的内容，只能缩放页面才行。

3.ideal viewport (理想视窗)

理想视窗没有固定的大小，理想视窗的宽和高，取决于移动设备的宽和高。iPhone6的屏幕尺寸是375px*667px(此处单位都是css像素），所以ideal viewport的宽度就是375px。在没有缩放的情况下，ideal viewport 的大小和visual viewport大小一致。

4.三者关系

在移动设备中viewport默认值是layout viewport,如果我们想要在移动端不需要滚动条或者缩放页面就能很好的查看页面，那么需要layout viewport=ideal viewport,那么该如何实现呢？

5.通过meta设置viewport把viewport设置为ideal viewport

下面是meta标签中viewport常用的一些设置：

属性 属性值 描述
width device-width 或者 正整数 定义layout viewport的宽度
initial-scale [0~10.0] 初始缩放值
minimum-scale [0~10.0] 最小缩放比例
maximum-scale [0~10.0] 最大缩放比例
user-scalable yes/no 是否允许用户手动缩放页面，默认是yes

方法1：通过width把viewport的值设置为ideal viewport

方法2:设置页面的缩放比例

这两种方法都可以把viewport设置为ideal viewport, 如果2种方法同时设置出现冲突例如content=’width=800,initial-scale=1’,浏览器会取2者中的最大值。

6.关于initial-scale

initial-scale是页面的初始缩放值，是viewport 相对于ideal viewport进行缩放。

以iPhone6为例：
当初始缩放比例 initial-scale = 1的时候，表示viewport 与 ideal viewport值相等都是 375px , 如果initial-scale = 0.5, 表示viewport相对于ideal viewport缩小了1倍，原来1px能展示的东西,现在需要用2px来展示，因此viewport为750px。如下图所示列：


关于viewport和ideal viewport有一个公式如下:

initial scall = ideal viewport 宽度 / visual viewport 宽度

兼容问题：
安卓自带的webkit浏览器只有在 initial-scale = 1 以及没有设置width属性时才是表现正常；IE浏览器initial-scale这个属性表现出来的效果永远是1。
