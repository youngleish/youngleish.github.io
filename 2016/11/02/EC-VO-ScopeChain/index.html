执行上下文和作用域

相关概念 ：

执行上下文(Execution Context = EC)：

为JS中执行的代码提供了环境，这个环境定义了代码在执行过程中可以获取的变量、函数等数据。（下面的介绍会统一使用EC表示执行上下文）

执行上下文栈(Execution Cotext Stack = ECS):

ECMScript有三种可以执行的代码：全局代码，函数代码，Eval代码（详解看这里）。每一种可执行的代码都会调用自身的EC，这一系列EC就会组成一个执行上下文栈,也就是ECS。

全局代码会产生全局执行上下文，在浏览器中，全局执行上下文被认为是Window对象，放在栈底。当代码执行流进入一个函数的时候，该函数的EC就被放到栈顶，当该函数执行完毕以后，EC就被销毁，全局执行上下文会在退出程序的时候，例如关闭浏览器时被销毁。

变量对象(Variable Object = VO):

每一个EC都会有一个与其关联的变量对象（下面介绍内容统一使用VO）,EC中定义的变量、函数、参数等都会存在这个对象中。如果是函数的EC,则把活动对象(Active Object = AO)作为VO，活动对象最开始时就会包含一个arguments对象。

关于AO中的arguments对象我有一个疑问，在《JavaScript高级程序设计》第三版中描述AO中的arguments属性值就是一个数组，数组里面是参数的值，但是网络上一些文章中AO的arguments属性值是一个json对象，对象里面包含caalle,length,和参数(下文截图中用的就是网络文章中的写法)，我不太确定那种说法对，目前持保留态度后续会确认更新。

作用域(Scope):

从字面意来看域是空间或者范围，那么作用域是提供了一个范围，那是为谁？又为了做什么提供的范围呢？

作用域是把所有的变量、函数、参数等收集起来组成查询对象，通过一些规则来确定执行代码有权限访这些标识符、函数等。

全局作用域：

  • 最外层函数和在最外层函数外定义的变量拥有全局作用域；
  • 未通过var声明的变量也自动拥有全局作用域
  • 所有window对象的属性拥有全局作用域

局部作用域：

  • 局部函数声明都是拥有局部作用域
作用域链(scope chain)：

关于作用域链我之前一直有一个错误的理解，认为作用域链是每个函数的作用域连在一起生成的一个列表，全部的执行代码中只会生成一条作用域链（诶,还是too young too simple）。
下面说一下我对作用域链最新的理解：

作用域链就是把各个EC中的VO/AO串联起来的有序列表，用来保证执行代码有序的访问变量、函数、参数等，他只是引用各个VO/AO并不是实际包含。

函数如何生成作用域链：

以函数为例，每一个函数在声明的时候就会创建一个包含着所有上层VO/AO的的作用域链,这个作用域链会被保存在一个函数内部的属性[[scope]]中。 当函数执行的时候会创建EC,解析器会复制函数[[scope]]属性中的对象构建EC中的作用域链。

作用域链的顺序：

假如全局EC中有一个function名为fn1内部嵌套了一个function名为fn2,fn2的作用域链顶端就是他自己的AO,下一级是fn2的父级函数fn1的AO,最后作用域链底端的是全局VO。每一个函数的作用域链都是按着这个顺序形成的，作用域链的顶端永远是当前函数的AO。

标识符解析：

如果某个函数执行过程用到一个变量,该变量在其EC中没有定义过，那么代码会沿着作用域链从最前端开始一级一级的搜索直到找到该变量为止。这个沿着作用域链搜索标识符（变量）的过程就叫标识符解析。

注意：内部EC中的代码执行过程可以通过作用域链访问外部EC的变量，函数等，但是外部EC中的代码不可以访问内部EC.

JS代码执行细节

当代码在EC中执行的时候，分为2个阶段：

  • 预解析（创建阶段）：

    • 创建作用域链

    • 解析器创建VO/AO(创建变量，函数，参数)

      • 根据函数的参数创建arguments对象
        • arguments:{
          callee: 函数名称,
          length: 函数长度,
          参数名: 参数值
          }
      • 扫描执行代码，查找函数声明
        • 多有函数声明都以函数名：函数引用的方式存入VO/AO中
        • 如果函数重名，则后面的会把前面的函数覆盖

      • 查找变量

        • 所有变量以变量名：undefined方式存入VO/AO中
        • 如果变量和函数或者参数重名，不会影响已经存在的函数或者参数

        • 如果同一个EC中后面多次声明同一个变量（未进行初始化），解析器只会解析第一个变量

        • var声明的变量会自动添加到最近的执行上下文中，如果变量初始化时没有通过var声明，js解析器会自动把变量添加到全局执行上下文中。

    • 确定this
  • 逐行解读代码（执行阶段）
    • 分配值，函数的引用，执行代码

hoisting 提升

提升包括函数声明提升和变量提升，函数声明是指函数执行之前解析器先读取函数声明。变量提升是指无论变量声明在EC的任何位置，解析器都会把变量声明提升到EC的顶部。如果函数声明和变量声明同时存在，会先提升函数声明后提升变量声明。

注意：关于变量提升需要注意的是，在预解析阶段只会提升变量的声明，不会提升赋值。
